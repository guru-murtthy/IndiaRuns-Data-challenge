================================================================================
MEMBER 4 HANDOVER DOCUMENT — INDIA RUNS HACKATHON
================================================================================

This document explains what Member 4 (the Orchestrator) has built and how the rest of the team should integrate their ML models.

--------------------------------------------------------------------------------
1. PROJECT STRUCTURE
--------------------------------------------------------------------------------
The project is strictly separated to avoid merge conflicts:

- `orchestrator/`: Member 4's complete FastAPI web app, data pipelines, and CSV generation code.
- `team_components/`: The home for your ML code.
  - `member1_parsing/`: Put your resume/JD NLP parsers here.
  - `member2_ranking/`: Put your FAISS index, embeddings, and LightGBM/CatBoost code here.
  - `member3_validation/`: Put your LLM/honeypot evaluation code here.

--------------------------------------------------------------------------------
2. HOW TO PLUG IN YOUR CODE
--------------------------------------------------------------------------------
Member 4 has created strict interface adapters. You do not need to modify the web app or the CSV exporter. You ONLY need to replace the logic inside these three files:

1. `orchestrator/adapters/member1_adapter.py`
   - Implement your own `parse_job_description()` logic to extract required skills and experience from raw JD text.

2. `orchestrator/adapters/member2_adapter.py`
   - Implement `rank_candidates()`.
   - Right now, it uses a dummy heuristic score. Replace this with your FAISS retrieval + LightGBM ranker.
   - Requirement: You MUST return a `semantic_score` and a `rank_score` for each candidate.

3. `orchestrator/adapters/member3_adapter.py`
   - Implement `validate_and_explain()`.
   - Replace the current dummy honeypot logic with your advanced validation.
   - Requirement: You MUST return a `validation_score`, a `final_score`, and a 1-2 sentence `reason` string explaining why the candidate was matched.

--------------------------------------------------------------------------------
3. RUNNING AND TESTING
--------------------------------------------------------------------------------
To test your integration, simply run the official command from the root directory:

    python rank.py --candidates data/raw/candidates.jsonl --out submission/outputs/test.csv

This will run the entire pipeline end-to-end. If it finishes without errors, your code works perfectly with Member 4's orchestrator!

To see the UI:
    uvicorn orchestrator.main:app --reload

Happy Hacking!
